A [[Ethernet]] comutada é uma rede de topologia em estrela com hubs ou [[Switch]] comutados como centro. Na Ethernet comutada, o [[Switch]] determina para qual porta do [[Switch]] o quadro de dados deve ser enviado de acordo com o endereço [[MAC]] no quadro de dados recebido. Os quadros transmitidos entre os nós são protegidos uns dos outros. Portanto, um nó não precisa se preocupar com a possibilidade de colisão de seus quadros de dados com os quadros enviados por outros nós ao passar pelo [[Switch]]. Ou seja, as interfaces pertencem a diferentes domínios de colisão.

Na Ethernet comutada, o [[Switch]] fornece um canal de informações dedicado para cada nó. A menos que duas portas de origem tentem enviar informações para a mesma porta de destino ao mesmo tempo, cada porta de origem pode se comunicar com sua respectiva porta de destino simultaneamente, sem colisão. Isso permite que cada nó use toda a largura de banda, em vez de compartilhá-la.

Devido às vantagens de reduzir conflitos e melhorar a largura de banda, a Ethernet comutada substituiu gradualmente a Ethernet compartilhada. O [[Switch]] só é diferente do hub no modo de trabalho, e outros modos de conexão, seleção de velocidade etc. são basicamente os mesmos do hub. Portanto, não há necessidade de alterar outros hardwares da rede original, inclusive cabos e NICs de usuário, ao substituir a [[Ethernet compartilhada ]]pela [[Ethernet comutada]]. Você só precisa substituir o hub tradicional por um hub ou [[Switch]] comutado, economizando assim o custo de atualizações de rede.
